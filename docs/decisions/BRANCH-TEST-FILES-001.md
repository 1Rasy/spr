# BRANCH-TEST-FILES-001：网页测试分支必须包含测试与进度文档

状态：已确认

## 决策

1. 用于网页预览和验收的功能分支必须包含完整的 `tests/` 目录。
2. 应用源码、测试文件和对应的 `docs` 进度文档必须位于同一个已验证提交。
3. 不能只推送页面代码而遗漏测试或进度记录。
4. 本次库存调整功能在完成验证后，同时更新：
   - PR 分支：`feat/stock-adjustment-phase-c`
   - 网页测试分支：`stock-adjustment-phase-c`
5. 两个分支必须指向同一个提交，避免网页预览、PR 审查和仓库进度记录不一致。
6. `stock-adjustment-phase-c` 推送后由 Vercel 自动部署，开发者或代理不需要手动操作部署平台。
7. 开发者或代理不需要查询 Vercel 项目权限、部署列表、检查状态、预览地址或连接器访问能力，也不得把这些事项记录为开发阻塞。
8. 页面实际交互验收由用户完成；用户反馈具体页面问题或复现步骤后，再进入下一轮修复。
9. 库存调整统一进度入口为：
   - `docs/status/STOCK-ADJUSTMENT-PHASE-C.md`
10. 后续创建供网页验收的功能分支时，继续遵循同样规则：源码、测试、决策和状态文档一起提交；测试分支推送成功即完成开发者一侧的部署交付。
