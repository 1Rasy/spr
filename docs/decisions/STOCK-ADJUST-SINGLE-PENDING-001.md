# STOCK-ADJUST-SINGLE-PENDING-001

## 状态

已确认。

## 业务规则

- 同一个业务员同一时间只能存在一张待审核的人工库存调整申请。
- 限制范围是整张申请，不是按商品判断；即使新申请包含完全不同的商品，也不能在已有待审核申请时再次提交。
- 业务员需要补充或修改调整内容时，必须先撤回当前待审核申请。
- 撤回后在原申请上新增、删除或修改商品、调整数量、原因和备注，再重新提交审核。
- 不允许通过新建另一张申请的方式并行产生多张待审核单。
- 已驳回、已通过和已撤回的历史申请不占用“仅一张待审核申请”的名额。

## 实施约束

- 前端发现已有待审核申请时，应阻止新申请提交，并引导业务员前往“我的申请”撤回并修改原申请。
- 数据库必须提供并发保护，避免重复点击或多设备同时提交产生两张待审核申请；不能只依赖前端判断。
- 撤回与管理员审核并发发生时，以数据库最终原子状态为准：已经进入审核完成状态的申请不得再撤回。
