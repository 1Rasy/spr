# STOCK-ADJUST-STOCK-FLOW-001

## 状态

已确认。

## 业务规则

- 人工库存调整申请只有在管理员审核通过后才改变正式库存。
- 每一项审核通过后实际发生的库存变化，都必须生成对应库存流水。
- 库存流水是人工库存调整的正式追溯依据，不能只修改 `van_stocks` 而不留流水。
- 一张申请包含多个商品时，每个商品分别生成一条库存流水，并通过同一个调整申请单号关联。
- 驳回、撤回、修改待提交和重新提交审核本身均不改变库存，因此不生成正式库存变化流水；相关过程只保留申请操作历史。

## 库存流水必须记录

- 调整申请单号；
- 商品；
- 调整数量；
- 调整原因；
- 操作人；
- 审核时间；
- 调整前库存；
- 调整后库存。

## 实施约束

- 调整前库存以管理员审核通过当下的最新库存为准。
- 调整后库存等于调整前库存加上申请中的正负调整数量。
- 正式库存更新、申请审核状态更新和库存流水写入必须在同一个数据库事务中完成。
- 任一商品库存更新或流水写入失败时，整张申请回滚，不得出现部分商品已调整、部分商品未调整，或库存已变化但流水缺失的情况。
- 已生成的库存流水不得直接修改或删除；发现错误时通过新的反向库存调整申请纠正，并生成新的库存流水。
