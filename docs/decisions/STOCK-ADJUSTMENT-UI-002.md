# STOCK-ADJUSTMENT-UI-002 库存修改入口与界面规则

- 状态：已确认并按最新反馈更新
- 首次确认日期：2026-07-12
- 最后更新日期：2026-07-12
- 适用范围：人工库存调整 Phase C 员工端
- 当前进度入口：`docs/status/STOCK-ADJUSTMENT-PHASE-C.md`

## 1. 入口位置与进入动画

库存修改申请入口必须放在员工“库存管理”页面内。

- 员工从开单首页进入“库存管理”；
- 在库存管理页点击“申请修改库存”进入修改模式；
- 不在开单首页顶层功能区单独增加“库存调整申请”入口；
- 点击后必须直接呈现库存修改页面；
- 不允许先隐藏库存管理页返回按钮、导致原页面上跳后再进入修改页；
- 实现时先替换目标页面内容，再切换页面级返回按钮可见性；
- 库存修改页只保留一个“返回库存查看”按钮；
- 管理端审核入口保持不变。

## 2. 页面结构顺序

库存修改页从上到下按以下顺序排列：

1. 返回库存查看；
2. 页面标题和说明；
3. 申请记录区域；
4. 品牌与规格筛选；
5. 商品列表；
6. 已选择商品摘要；
7. 底部“保存并提交审核”按钮。

申请记录必须位于商品筛选和商品列表上方，并拆分为：

- 待审核申请；
- 已驳回申请；
- 未提交草稿；
- 历史记录；
- 已撤回申请。

待审核和已驳回不得合并成一个区域。

## 3. 视觉与商品卡片

库存修改界面必须与现有开单页、库存管理页保持一致。

- 复用 `store-style.css` 的页面容器、返回按钮、品牌标签、规格标签、商品卡片、数量控件和底部提交按钮；
- 可使用 `store-stock-adjustment.css` 补充库存修改专用布局，但不得重新制作一套割裂的视觉体系；
- 商品排序、品牌切换、规格切换沿用开单/库存管理逻辑；
- 商品名称下方不再重复显示类似“97g 原味”的规格口味文字行；
- 当前库存和预计库存必须采用相同左侧基线对齐；
- 页面标题、按钮和提示文案使用中文。

## 4. 数量修改方式

员工只修改散数。

- 不提供箱数、盒数、整件或价格控件；
- “增加”和“减少”使用直接按钮，不使用方向下拉框；
- “增加”在上、“减少”在下，竖向放在散数数量控件左侧；
- 选中方向必须有明显状态；
- 散数必须使用与开单页相同的 1–25、5×5 数量弹窗；
- 数量弹窗提供当前数量、1–25、清零和关闭操作；
- 散数必须为非负整数，最终提交时转换为带符号的最小库存单位；
- 增加提交正数，减少提交负数；
- 允许预计库存为负数；
- 修改已驳回或已撤回申请时，必须恢复原方向和绝对值散数。

商品行推荐结构：

```text
商品名称
当前库存：...
预计库存：...

[增加]  散 [数量] 单位
[减少]
```

## 5. 局部更新

- 修改单个商品方向或数量时，只更新当前商品行和已选摘要；
- 不重新加载申请记录；
- 不重建整个页面；
- 不改变滚动位置；
- 品牌或规格切换时，只允许重绘商品区域；
- 提交、撤回、编辑完成后，才重新请求申请记录。

## 6. 提交流程与原因

页面主体不提前展示原因和备注输入框。

点击“保存并提交审核”后，才打开提交面板，员工在此时选择原因并填写备注。

原因选项只保留：

- 盘点差异；
- 破损报废；
- 调货；
- 其他。

规则：

- 删除“漏录入库”；
- 选择“其他”时必须填写原因说明；
- 备注可选；
- 提交失败时保留已选商品、方向和数量；
- 提交成功后清空本次调整并返回库存查看。

## 7. 验收标准

- 开单首页不单独显示库存调整入口；
- 从库存管理页进入库存修改时没有原页面上跳或闪动；
- 修改页只出现一个返回按钮；
- 商品名称下方不出现重复规格口味行；
- 增加、减少竖排在数量左侧；
- 数量点击后出现 5×5 数量弹窗；
- 当前库存与预计库存左对齐；
- 申请记录置于页面上方并按状态分区；
- 原因只在提交时选择，且没有“漏录入库”；
- 只能提交整数散数；
- 员工端不直接更新 `van_stocks`，仍通过申请和审核流程完成库存变化。
