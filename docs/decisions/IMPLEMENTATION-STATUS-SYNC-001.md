# IMPLEMENTATION-STATUS-SYNC-001：代码与进度文档必须同步

- 状态：已确认
- 确认日期：2026-07-12
- 适用范围：由 Codex、ChatGPT 或其他开发者继续实施的功能分支

## 决策

任何直接修改代码、数据库 migration、测试或页面交互的工作，都必须在同一次推送中同步更新对应的仓库内进度文档。

库存调整 Phase C 的统一进度入口为：

`docs/status/STOCK-ADJUSTMENT-PHASE-C.md`

## 每次更新必须记录

1. 本次完成的功能或修复；
2. 尚未完成的问题；
3. 修改的核心文件；
4. 实际执行的测试命令、范围和结果；
5. 数据库 migration 是否已部署；
6. PR 分支与网页测试分支是否指向同一提交；
7. 当前 PR 是否仍为 Draft、是否已合并；
8. 页面实际验收是否已有用户反馈；没有反馈时不得自行把部署平台访问问题写成阻塞。

## 不可替代项

以下内容只能作为补充，不能替代仓库内进度文档：

- 聊天记录；
- PR 描述；
- PR 审查评论；
- 自动部署状态；
- 临时提示词。

## 执行规则

- 代码已经完成但状态文档未更新，视为任务未完整交付；
- 状态文档不得保留已经失效的限制或旧结论；
- 后续人员开始工作前，应先读取状态文档和最新决策文档；
- 每轮完成后，将应用源码、测试文件和文档同步到同一个已验证提交；
- `feat/stock-adjustment-phase-c` 与 `stock-adjustment-phase-c` 必须保持一致；
- 测试分支推送后由平台自动部署，开发者无需操作或排查 Vercel；页面验收由用户完成。
